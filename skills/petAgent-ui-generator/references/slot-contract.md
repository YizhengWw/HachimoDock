# COMPONENT_DASHBOARD_V1 槽位契约

设备端 800×480 LCD 用固定布局画 9 个槽位。每个槽位有 UTF-8 字节上限(超出会被设备端 br_normalize_text 安全截断,但视觉会糟糕,所以生成时就要控好)。

## 9 槽位

| 槽位 id | 最大字节 | 设备版式角色 | 推荐写啥 |
|---|---|---|---|
| `title` | 60 | 左上角徽章 | 组件名(2-6 个汉字) |
| `eyebrow` | 90 | 徽章下小号说明 | "距离 X" / "当前 X" / 上下文标签 |
| `headline` | 156 | 大号高亮句 | 主信息(最显眼的一行) |
| `metricLabel` | 90 | 指标面板标题 | "剩余时间" / "下班时间" / "间隔" 这类 |
| `metricValue` | 60 | 大号数值 | 数字 / 时间 / 状态文字 |
| `metricUnit` | 30 | 数值后小号单位 | "分钟" / "TOKEN" / "" |
| `badge` | 12 | 右上角绿色圆数字 | 计数(轮次 / 次数,1-3 字符为佳) |
| `note` | 156 | 指标下方小号说明 | 附加信息(本轮目标,本周累计等) |
| `footer` | 156 | 底部小号提示 | "点击 X · 长按 Y" 触摸操作提示 |

## CJK 字节预算速算

中文 1 字 = 3 字节。所以:
- title 60 字节 = 最多 20 个汉字 / 60 个 ASCII
- badge 12 字节 = 最多 4 个汉字 / 12 个 ASCII(实际推荐 1-3 字符)
- headline 156 字节 = 最多 52 个汉字

## payload 格式(供参考)

设备端从 `.stats-display` 读文本,首行必须是 magic:

```
COMPONENT_DASHBOARD_V1
title=摸鱼倒计时
eyebrow=距离今天下班
headline=还有 2 小时 13 分
...
```

未提供的槽位会被忽略(空显示)。`COMPONENT_DASHBOARD_V1` 是 22 字节,parser 用 strncmp 严格匹配。

## 不允许的事

- 不要发明新槽位 — `negative-screen.json.dashboard` 里的 key 只能是上面 9 个
- 不要在槽位值中嵌入换行符(payload 是按 `\n` 分行的)
- 不要让用户依赖 `metricValue` 里的复杂格式(设备 UI 会按需缩小字号,但太长就丑)

## Pet Manager 组件中心按钮功能控件

| 控件 | event 字符串 | 物理动作 |
|---|---|---|
| 屏幕区域 | `screen.region.tap` | 触屏点击 |
| 屏幕区域 | `screen.region.long_press` | 触屏长按 |

每个组件至少说明切到负一屏后的默认场景 / 默认状态,以及点击和长按分别做什么；如果用户要求红钮或旋钮,说明这些硬件不用于组件动作,旋钮固定为系统音量。如果用户没说清,先追问。footer 槽位写操作提示,例如 `"点击 开始/暂停 · 长按 重置"`。
