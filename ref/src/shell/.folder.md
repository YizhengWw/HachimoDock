# Folder Plan: ref/src/shell

## Architecture
- Scope: ref/src/shell
- Function: Cross-page shell primitives that compose every page in the manager — PageShell (header + actions + help), Card (with Collapsible variant), ToastStack (provider + hook + presentational stack), DeviceContextProvider (single source of polling for USB status, independent WiFi availability, binding/appearances/agents; derives overall deviceOnline from USB-or-WiFi without collapsing the two status channels; hydrates active channel from the bridge profile before trusting localStorage so stale UI cache cannot mark an unsynced appearance as current; owns the in-flight appearance USB sync progress so dashboard tab unmounts do not reset the visible downlink task; reconciles online board `targetSource` back to the selected client channel when availability is stale; Rust owns background serial auto-connect; exposes manual USB serial rescan/connect for user-triggered recovery), ContextRail (sidebar bottom rail showing the device/appearance/component triad).
- Sync: if this folder changes, update this file immediately.
- Reuse: consumed by `App.jsx` (provider wiring + sidebar mount), and by every page in `ref/src/*.jsx` after Plans 2/3/4 migrate them. Pure derivations like `deriveCurrentDisplay` are unit-tested via `node:test`.
