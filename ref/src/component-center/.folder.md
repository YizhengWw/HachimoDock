# Folder Plan: ref/src/component-center

## Architecture
- Scope: ref/src/component-center
- Function: Sub-components for the redesigned Component Center page — NowShowingHero (top "现在装着什么" card), component library grid, and ComponentPreviewModal. These are composed by `../ComponentCenter.jsx` (wired in Step 3).
- Sync: if this folder changes, update this file immediately.
- Reuse: Card imported from `../shell/Card.jsx`; CSS tokens from `../styles.css`.

## Files
| File | Pos | Function |
|---|---|---|
| `NowShowingHero.jsx` | `component` | Top hero card showing the currently installed component: device-screen preview, chip badges, button-mapping list, and 更换 button. Empty state is passive copy only because the library grid is already directly below. |
| `NowShowingHero.test.js` | `test` | Static Node coverage (8 tests) for default export, Card import, bound/empty class names, device-screen reuse, bindings map, disabled guard, and no redundant 浏览组件库 CTA in empty state. |
| `DeviceScreenPreview.jsx` | `component` | Shared negative-screen preview renderer that emits the fixed-slot component-device-screen DOM tree used by built-in/draft cards, current-component hero, and install modal. Progress renders in its own bottom slot so card previews stay visually consistent across widgets. |
| `DeviceScreenPreview.test.js` | `test` | Static Node coverage for default export, root class, title badge, separate bottom progress slot, progress normalization guard, and CSS frame stability. |
| `CandidateCard.jsx` | `component` | Compact grid card for the component library: adaptive complete mini device-screen preview that follows the card frame, builtin/draft source modifier, name, builtin/custom badge, full goal blurb, and a draft-only delete button that calls the parent delete confirmation without opening ComponentPreviewModal. |
| `CandidateCard.test.js` | `test` | Static Node coverage (10 tests) for default export, prop signature including `onDelete`, candidate-card class, badge/source variant logic, draft-only independent delete action, component-device-screen reuse, shared adaptive builtin/draft/create card geometry CSS, and complete preview/copy display. |
| `ComponentPreviewModal.jsx` | `component` | Full-screen final install-confirm modal: large device-screen preview, button-mappings list, replace-warning banner, offline-warning banner, and Install / Cancel footer with disabled states; parent `../ComponentCenter.jsx` starts OTA directly from this button. |

| `binding-labels.js` | `lib` | Binding-label model: `formatBindingControl`/`isRoutedWidgetBinding` (rich status labels) plus the install-time `CONTROL_OPTIONS` vocabulary and `bindingKey`/`defaultControlLabelForBinding`/`optionForControlLabel` helpers `../ComponentCenter.jsx` uses to resolve a binding label. Pure JS, no JSX. |
| `binding-labels.test.js` | `test` | Behavior coverage (bare node) for both label vocabularies, the CONTROL_OPTIONS bindable list, and the resolution helpers. |
| `draft-utils.js` | `lib` | Pure (no-JSX) draft helpers: `buildDraftGoal` summary copy and `matchesDraftPath`/`normalizeLocalPath`/`pathContainsComponentId` cross-separator clawpkg-path matching, extracted out of ComponentCenter. |
| `draft-utils.test.js` | `test` | Behavior coverage (bare node) for draft summary copy and Windows/POSIX path matching. |
