## Architecture
- Scope: ref
- Function: Runnable Tauri 2 desktop client for configuring the shared petAgent bridge profile, device setup, dashboard, pet album, appearance/logo assets, a four-source custom appearance entry (图片AI生成 / 自定义上传视频 / Codex 导入 / 社区导入), direct MP4-upload custom appearances, compressed per-state appearance WAV prompt sounds with default done/error/waiting_user fallback and user overrides, a preview-first appearance detail workspace with sticky current-state controls plus a horizontal all-known-state rail, direct per-state MP4 replacement, custom appearance single-state image+prompt regeneration through a closable detail-page modal that reuses the full avatar-generation upload/config step UI and continues generation in the background, board voice controls, avatar generation that skips importer-only directional `touch.right` / `touch.left` states for user-created appearances, and a responsive component center for negative-screen widgets. Dashboard voice controls use a single-column layout with per-button function editing for voice and negative-screen actions, mock voice-text injection into the active agent session, board-originated voice action status, selected-agent-only active USB/speech forwarding with SSH state fallback only when USB is disconnected, multi-session Codex and Claude Code speech publishing with duplicate suppression, and full visible button-map USB `button_config` OTA where the Tauri command waits for board `button-config-ack` and retries once through a USB reconnect if the serial writer is stale before the UI reports success. Appearance asset USB OTA is transactional and host-serialized: full MP4/WAV sync and audio-only patch sync share one desktop-side guard before `asset/begin` so they cannot interleave inside the board's global `.incoming-desktop-pet` staging; each MP4/WAV file carries size/checksum metadata and waits for board `asset/ack` on `asset/file`, then waits for final commit ack before reporting success; when videos already match, changed WAV cues use the board `asset/patch-commit` audio patch path, while removed local cue files force a full sync so stale board WAVs are cleared; single-state regeneration or direct MP4 upload replaces only the selected family MP4 in the local manifest and then reuses this same appearance asset OTA to replace it on the board; if a board times out on newer `asset/stat`, per-file `asset/file`, or `asset/patch-commit` phases, the host falls back to a legacy full sync that only relies on begin/chunk/commit. Widget (negative-screen) USB OTA is now also ack-gated: client registers begin/commit waiters and waits for board `widget-install-ack` so the success toast lines up with the device actually activating the new widget, with graceful timeout fallback for legacy boards. Widget SSH OTA collapses the old 3-stage `tar→scp→ssh-extract` (~1.8s of fixed handshake) into a single `tar -czf - | ssh '… tar xzf -; mv'` streaming pipe so a ~3KB widget no longer pays 100% handshake overhead. Component center supports coding-agent component generation through `petAgent-ui-generator`, generated draft install/delete flows, replacement confirmation, and per-component button-function customization across screen touch, top red button, and front encoder that is downlinked through USB/SSH `.clawpkg` installs.
- Sync: if this folder changes, update this file immediately.
- Reuse: the React frontend stays in `src/`, while the Tauri runtime in `src-tauri/` owns shared bridge profile storage, local bridge lifecycle management, stale legacy bridge cleanup, packaged-resource bridge selection with debug-only source-tree fallbacks, bundled bridge resources, Windows-safe bridge/voice-service process spawning with bundled Node, Codex pet scanning/import commands, controlled Codex Pets CLI installs, the flattened Rust-side HTTP bridge used by desktop avatar generation, MQTT remote-binding dispatch for wireless channel following with stale-USB fallback protection, stale-state-safe USB-first status forwarding with disabled-source clearing, active speech sync, and SSH runtime-file fallback sourced from env or local pet-screen config only after USB disconnects, persisted desktop-device identity propagation into the USB handshake, macOS USB CDC/USB-UART scan normalization that accepts `usbmodem`, `usbserial`, and common adapter VIDs while preferring `/dev/cu.*` callout ports over blocking `/dev/tty.*` pairs, USB reconnect cancellation that releases stale reader clones before reopening a locked callout port, generated component draft discovery with manifest descriptions for UI card summaries, explicit Codex session-dir passthrough for temporary-HOME walkthroughs, managed bridge launchers that pin `CLAWD_BRIDGE_PORT` to 23333, prepend the selected Node runtime directory to `PATH` so LaunchAgent-started CLI shims can run, auto-restart the managed bridge when the selected Codex adapter is recoverably unhealthy, and pass an account-safe Codex model override, Codex/Claude voice session listing that prefers client-provided names over bootstrap context, skips Codex rollout JSONL files without `session_meta`, keeps `auto` resume on the intended named session, and retries mock/board voice injection with a fresh Codex session when stale rollout metadata prevents resume, and the constrained external URL opener now limited to http(s) community links.
